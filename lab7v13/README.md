# Лабораторна робота №7: Патерн Retry та обробка IO/мережевих помилок

## Варіант №13: Читання даних з сенсора

### 📝 Опис проєкту
Проєкт демонструє реалізацію відмовостійкої системи для збору даних з локальних файлів та віддалених API. Основна увага приділена обробці тимчасових помилок за допомогою стратегії повторних спроб (Retry).

### 🏗 Реалізація патерна Retry
Використано узагальнений метод `ExecuteWithRetry<T>`, який підтримує:
* **Експоненційну затримку:** Час очікування подвоюється з кожною невдалою спробою.
* **Вибіркову обробку:** Через делегат `Func<Exception, bool>` можна вказати, на які саме типи помилок варто реагувати.
* **Логування:** Програма виводить у консоль інформацію про кожну невдалу спробу та причину.

### 🧪 Тестові сценарії
1. **Локальний сенсор:** Імітується `IOException` (файл зайнятий). Успіх на 3-й спробі.
2. **Мережевий сенсор:** Імітується `HttpRequestException` (сервіс недоступний). Успіх на 5-й спробі.
