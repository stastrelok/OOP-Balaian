Короткий звіт до лабораторної роботи №24
1. Патерн Strategy:
Ми реалізували інтерфейс INumericOperationStrategy. Це дозволило зробити клас NumericProcessor незалежним від конкретних математичних формул. Ми можемо додавати нові операції (наприклад, FactorialOperation), не змінюючи код процесора, що відповідає принципу OCP (Open/Closed Principle).

2. Патерн Observer:
Використання подій C# (event Action<...>) дозволило реалізувати механізм "один до багатьох". Коли обчислення завершується, ResultPublisher сповіщає всіх підписників. При цьому він не знає, хто саме на нього підписаний (консоль, історія чи алерти), що забезпечує низьку зв'язність (Low Coupling).

3. Висновок:
Комбінація цих патернів робить систему надзвичайно гнучкою: ми можемо динамічно змінювати спосіб обробки (через Strategy) та спосіб реакції на результат (через Observer), не втручаючись у внутрішню логіку компонентів.
